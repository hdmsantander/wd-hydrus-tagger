    Performance (CPU / GPU inference)

    Saved to config.yaml. Inference batch size applies to every tagging run. Changing CPU thread counts reloads the model on the next run.

    Automatic server tagging configuration

    Tag all writes each inference batch to Hydrus as it runs. These options apply only to Tag selected. WebSocket progress uses batch summaries; the queue bar still shows during a run.

    Use 1–256. Smaller N pushes to Hydrus more often.

    This stride applies only to automatic server tagging over WebSocket. It is not the same as files per apply request under Apply all tags (HTTP).

    When push is off, nothing is written to Hydrus during the run; tags accumulate in the results list. After the run, review and edit them there, then use Apply all tags to Hydrus. When push is on, choose a tag service before you start.

    Session auto-tune (Tag all)

    Tag all only. After warm-up, the server searches batch size, Hydrus parallelism, and optionally ONNX CPU intra-op threads (one phase at a time). This is online measurement over discrete knobs — not ML training. When auto-tune is enabled, the progress panel always includes the performance timing overlay for each batch.

    Supervised mode shows Approve tuning step when a new knob value is proposed (controller tab only).

    Splits the queue into a learning prefix and a commit suffix. While processing the prefix, incremental Hydrus writes are deferred so knobs can be explored; tags flush when the commit segment starts. Requires session auto-tune. The overlay shows Learning vs Commit and live estimates.

    Diagnostics (ONNX Runtime)

    Session profiling is off by default. When enabled, the next model load starts recording; trace files are written when the ONNX session ends (switch model, unload, or server stop). Expect slower inference and large files under the trace directory.

    To force profiling without the checkbox, set WD_TAGGER_ORT_PROFILING=1 (or true / yes / on) before starting the server; that overrides saved config until unset. See README → ONNX Runtime session profiling.

    WD model marker (skip & dedupe)

    Files already tagged with the current model marker can skip ONNX. After inference, a marker tag can be appended so later runs skip. Empty template uses wd14:{model_name}.

    When you re-tag with a larger WD model, images may already carry tags from a smaller model. Hydrus keeps existing tags; new predictions are added on top. The marker prefix option drops stale wd14: markers from the proposal so only the current model's marker remains after a run (merged libraries stay consistent).

    Apply all tags (HTTP)

    Chunk size for “Apply all tags to Hydrus” on the results screen (server splits large requests). Separate from Write to Hydrus every N in Automatic server tagging.
